Idea para abrir la clase

"La UI no conoce HTTP. Solo conoce estado. Todo lo demas ocurre en capas intermedias".

Mensaje clave sobre suspend

suspend no significa "corre solo en background"; significa "puede pausar la coroutine sin bloquear el hilo principal".

Mensaje clave sobre Coil

Retrofit trae datos JSON. Coil resuelve otro problema: descargar, cachear y pintar imagenes.

Mapa de capas

El flujo principal va de izquierda a derecha hasta la API. El camino de Coil es distinto: nace en la UI cuando ya existe una imageUrl.

UI

Compose Screen

CharactersList observa estado y renderiza loading, lista o error.

  • No hace HTTP.
  • No parsea JSON.
  • Solo dibuja estado.
Presentation

ListViewModel

Dispara la carga con viewModelScope.launch y expone el estado de la pantalla.

Domain

GetCharactersUseCase

Encapsula la accion "obtener personajes" y deja una puerta clara para reglas de negocio.

Data

CharacterImpl

Crea Retrofit, ejecuta la llamada y transforma la respuesta hacia el modelo que necesita la UI.

Infra

Retrofit + API

CharactersApi define el endpoint con anotaciones. Retrofit genera la implementacion real.

Remoto

Rick and Morty API

Devuelve JSON. Gson lo convierte en CharactersResponse y despues se hace el mapping a dominio.

Media

Coil

Toma la imageUrl, descarga la imagen y maneja loading y error.

Pregunta para el aula

Si la API falla, que deberia cambiar en la UI: seguir mostrando loading o pasar a un estado de error?

Flujo de datos

UI -> ViewModel -> UseCase -> Repository -> Retrofit -> API

Flujo de imagenes

UI -> Coil -> recurso remoto de imagen

Objetivo didactico

Separar responsabilidades para que cada capa tenga un motivo claro para existir.

Recorrido paso a paso

Cada paso activa nodos y conexiones. Te sirve para narrar la comunicacion entre capas y para remarcar donde aparece cada concepto.
