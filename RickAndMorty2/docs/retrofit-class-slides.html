¿Que hace esta app?

Un ejemplo simple, pero muy util para explicar responsabilidades.

02
  • Lista personajes de Rick and Morty.
  • Consume una API REST.
  • Muestra nombre, estado e imagen.
  • Usa Compose para pintar la pantalla.
La gracia del ejemplo no esta en la complejidad funcional, sino en que cada concepto aparece claro.
Mensaje para tus alumnos

Aunque la app sea simple, no queremos que la UI conozca detalles de red, parsing o carga de imagenes.

Flujo general

La informacion cruza varias capas antes de llegar a la pantalla.

03
Recorrido principal
MainActivity → ListViewModel → GetCharactersUseCase → CharacterImpl → CharactersApi → API
Camino paralelo
  • La UI recibe imageUrl.
  • Compose delega esa URL a Coil.
  • Coil descarga y cachea la imagen.
Acá conviene contrastar: JSON por un lado, imagenes por otro.

¿Que es una suspend function?

Una idea clave para entender coroutines y red.

04
  • Es una funcion que puede pausarse y reanudarse.
  • No bloquea el hilo principal.
  • Se usa mucho en operaciones de red.
  • Se ejecuta dentro de una coroutine.
Frase corta para repetir: suspend no es background automatico; es pausa sin bloqueo.
Ejemplo 
suspend fun getCharacters(): Response<CharactersResponse>

¿Donde aparece suspend en este proyecto?

No esta en un solo lugar: atraviesa varias capas.

05
1
En CharactersApi, porque el endpoint puede tardar en responder.
2
En CharacterImpl.getAll(), porque delega la llamada remota.
3
En GetCharactersUseCase.execute(), porque coordina el trabajo asincronico.
Uso con IO 
suspend fun execute(): List<Character>? =
    withContext(Dispatchers.IO) {
        characters.getAll()
    }

¿Que hace el ViewModel?

Es el intermediario entre pantalla y datos.

06
  • Guarda el estado de la pantalla.
  • Lanza la carga de datos.
  • Expone datos observables a la UI.
  • Evita que la UI conozca Retrofit.
La UI no dice “haceme un GET”. La UI observa un estado y renderiza.
Ejemplo 
viewModelScope.launch {
    _characters.value = getCharactersUseCase.execute()
}

Retrofit

Una interfaz declarativa para describir endpoints HTTP.

07
  • Define APIs con interfaces y anotaciones.
  • Genera la implementacion automaticamente.
  • Hace la llamada HTTP.
  • Se integra con converters como Gson.
Endpoint 
interface CharactersApi {
    @GET("character")
    suspend fun getCharacters(): Response<CharactersResponse>
}

Builder de Retrofit

Configura la base URL y como se convierte el JSON.

08
  • baseUrl le dice adonde apuntar.
  • addConverterFactory define como parsear la respuesta.
  • create() materializa la interfaz de la API.
Retrofit no trabaja solo: necesita configuracion explicita.
Configuracion 
private val retrofit = Retrofit.Builder()
    .addConverterFactory(GsonConverterFactory.create())
    .baseUrl("https://rickandmortyapi.com/api/")
    .build()

DTO, modelo y mapping

La UI no deberia depender del formato remoto.

09
  • CharacterDTO representa el JSON.
  • Character representa lo que usa la UI.
  • toModel() transforma una capa en otra.
  • Esto reduce acoplamiento.
Idea pedagogica

Mostrar esta separacion ayuda a explicar por que no conviene que la UI conozca nombres exactos del JSON.

¿Que hace Coil?

Carga imagenes remotas dentro de Compose.

10
  • Recibe una URL remota.
  • Descarga la imagen.
  • Maneja loading y error.
  • Usa cache y simplifica la UI.
Retrofit trae datos estructurados. Coil trae imagenes. Son responsabilidades distintas.
Ejemplo 
SubcomposeAsyncImage(
    loading = { LoadingBox() },
    error = { ImageError() },
    model = character.imageUrl,
    contentDescription = null
)

Estados de UI

Un punto importante para mejorar el ejemplo y enseñar buenas practicas.

11
  • Loading
  • Success
  • Error
Hoy el proyecto mezcla loading y error usando null. Para enseñar mejor, conviene separar estados.
Pregunta para el aula

Si falla la API, ¿deberia seguir viendose el skeleton de carga o un mensaje de error con retry?

Resumen final

Consumir una API es mas que hacer un GET.

12
Ideas que deberian quedar claras
  • La UI observa estado.
  • El ViewModel coordina la pantalla.
  • suspend permite esperar sin bloquear.
  • Retrofit consume la API.
  • Coil resuelve imagenes remotas.
Cierre sugerido

La clave no es solo consumir una API; la clave es separar responsabilidades para que el codigo sea entendible, mantenible y facil de enseñar.
